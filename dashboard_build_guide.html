01 How it fits together

You almost never touch Notion to enter data — you build the views and then read them.

You message the Telegram bot ("800 dinner, me and Aditya"). Apps Script parses it, resolves each name to a person, works out the split, records it in the Expenses database, and — when it can — pushes it to Splitwise. A background sync runs every ~15 minutes: it pulls your Splitwise activity (group and direct friend expenses) back into Notion, refreshes everyone's balances, and carries out anything you've flagged (settle a parked expense, merge a name, delete a row).

Notion's API can create databases and properties but not views — so the databases below already exist and stay in sync, and the view layer is the one thing you build by hand. Do it once; it keeps working.

02 Notion mechanics, in 60 seconds

Everything below is a linked view plus a filter. If you've built one before, skip ahead.

  1. On your dashboard page type /linked and choose Linked view of database, then pick the source database (e.g. Expenses).
  2. Open the view's ⋯ menu → Filter / Sort / Group and apply the recipe's spec.
  3. For sums: hover just below a number column's last row — a Calculate ▾ toggle appears; choose Sum. In grouped views, every group gets its own sum.
  4. ⋯ → Properties controls which columns show; drag to reorder.

08 🧩 How to resolve things

The handful of actions you'll actually take — all done by editing a row, no IDs to type.

Set up a person (clear a "Needs mapping" expense)

When: a new name appears, or an expense is parked as Needs mapping.

  1. Open the person's row in People.
  2. Glance at Candidates — the sync suggests likely contacts there — then pick them from the Splitwise Identity dropdown. Their Splitwise ID and readable name fill in on the next sync.
  3. Optional: set a Default Group to route their expenses into a specific Splitwise group. Leave it empty and they settle as direct friend expenses instead.
  4. Done. The next sync pushes any parked expenses with them and clears the flag — /sync in Telegram if you don't want to wait.

Merge a nickname or duplicate

When: the bot created a stray person (e.g. "Mang") that's really someone you already have (Mangalik).

  1. Open the stray person's row.
  2. Set Merge Into to the real person.
  3. The next sync folds them together: it saves the stray name as an alias, re-points every expense to the real person, and archives the stray. Future messages using that nickname resolve on their own.

Answer an ambiguous name

When: a name could be two people (e.g. "Adi" → Aditya or Aditi).

  1. The bot asks right in Telegram with buttons — tap the right person.
  2. That saves the nickname to them, so it's never asked again. Nothing to do in Notion.
  3. If the buttons expired: the expense is already logged — fix it with Merge Into here in Notion. Don't re-send the message; that would log it twice.

Delete or re-push an expense

When: the bot got something wrong, or you want to remove an expense everywhere.

  1. Fastest: tap 🗑️ Delete on the bot's reply in Telegram — it removes the expense from Splitwise and archives it in Notion, and tells you honestly if either step failed.
  2. From Notion: set Sync Action to Delete (removes it everywhere) or Re-push (rebuilds the Splitwise side — the replacement is created before the old one is deleted, so a failed re-push destroys nothing). The next sync carries it out and writes the outcome to Sync Status.

09 Settlement status reference

The Settlement Status on each expense tells you where it stands.

StatusMeaningDo you need to act?
Settled-via-Splitwise Pushed to Splitwise (a group or a direct friend expense). Balances reflect it. No.
Needs mapping Recorded, but a participant isn't set up yet, so it couldn't be pushed. Yes — set up the person (section 08). It clears automatically.
Notion-only Kept in your ledger but never sent to Splitwise — a solo expense, or historical data. No.

Views are a Notion UI feature the API can't create — building them by hand is a one-time setup. Once done, everything stays current on its own: new expenses land from Telegram, balances refresh from Splitwise, and your edits (identity picks, merges, deletes) are carried out on the next sync (~15 min — or /sync now, /status for health).
